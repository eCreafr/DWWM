Comment fonctionne ce layout ?

L'effet Pinterest (aussi appelé masonry = "maçonnerie") repose sur CSS Multi-Column Layout, une propriété CSS native, sans JavaScript.

  • column-width: 260px — le navigateur calcule automatiquement le nombre de colonnes qui tiennent dans la largeur disponible. Sur un grand écran → 4 colonnes ; sur mobile → 1 seule. C'est le comportement responsive intégré.
  • column-gap: 16px — gouttière entre les colonnes, identique à gap en Grid ou Flexbox.
  • break-inside: avoid — propriété essentielle à placer sur chaque carte. Sans elle, une carte peut être "fragmentée" : la moitié apparaît en bas d'une colonne et l'autre moitié en haut de la suivante.

Limite à connaître : les éléments sont distribués verticalement colonne par colonne (comme un journal), pas horizontalement ligne par ligne. L'ordre de lecture est donc : 1→2→3 dans la colonne 1, puis 4→5→6 dans la colonne 2, etc. Si l'ordre de lecture gauche→droite est critique (ex : timeline), préférez CSS Grid avec grid-template-rows: masonry (encore expérimental en 2025) ou une solution JavaScript.
