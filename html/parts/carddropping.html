Scroll-driven
+ @starting-style

Deux animations CSS en une : fade-in à l'apparition, puis motion lié au scroll.
Le secret ? Une propriété custom --progress pour découpler les deux effets.

🎯

Le problème

Animer opacity directement avec un scroll-timeline écrase l'état défini par @starting-style. Les deux effets entrent en conflit.

Contexte
🔑

Déclarer @property

On enregistre une propriété typée pour que le navigateur puisse l'interpoler.

@property --progress {
  syntax: "<number>";
  inherits: false;
  initial-value: 0;
}
@property
✨

@starting-style

Définit l'état de l'élément avant son premier rendu. Le navigateur transite depuis cet état dès l'insertion dans le DOM. 

@starting-style {
  .card { opacity: 0; }
}
@starting-style
📜

Connecter le scroll

On lie l'animation à la position de la carte dans le viewport.

  • animation-timeline: view()
  • animation-range: entry 0% cover 60%
  • animation: progress linear both
scroll-driven
🔗

Les effets visuels

Tout lit --progress au lieu d'être animé directement.

opacity: var(--progress);

transform:
  translateY(calc(
    (1 - var(--progress)) * 120px
  ))
  scale(calc(
    0.82 + var(--progress) * 0.18
  ));

filter: blur(calc(
  (1 - var(--progress)) * 10px
));
var()
📐

animation-range

Délimite la fenêtre d'animation dans le viewport. entry 0% = le bord bas de la carte entre dans le viewport. cover 60% = la carte couvre 60 % du viewport.

animation-range
🏗️

Le patron complet

1. @starting-style — état invisible avant le premier rendu.

2. @property --progress — valeur interpolable par le moteur CSS.

3. animation-timeline: view() — le scroll pilote --progress.

4. opacity / transform / filter lisent la valeur custom.

Pattern
🚀

CSS natif, zéro JS

Aucun IntersectionObserver, aucune librairie. Tout repose sur des fonctionnalités CSS modernes, performantes et composables.

No JS
⚡

Pourquoi ça fonctionne

@starting-style agit sur opacity à l'instant zéro. Le scroll-timeline n'anime jamais opacity lui-même, seulement --progress. Pas de conflit. 

/* scroll anime ceci */
@keyframes progress {
  from { --progress: 0; }
  to   { --progress: 1; }
}

/* pas opacity directement */
Mécanique
🌐

Support navigateur

Les trois briques sont disponibles dans les navigateurs modernes :

  • Chrome 115+ / Edge 115+
  • Firefox 110+ (@property)
  • Safari 17.2+ (@starting-style)
  • Safari 18+ (scroll-driven)
Compat
🎨

Autres propriétés pilotables

--progress peut alimenter n'importe quelle propriété : border-color, background, letter-spacing, clip-path…

Créatif
💡

Plusieurs propriétés, un seul scroll

On peut déclarer plusieurs propriétés custom et les animer sur des plages différentes avec animation-range.

Par exemple : --enter de entry 0% à cover 40%, et --highlight de cover 40% à exit 0%. Deux effets distincts sur la même carte, sans JS.

Avancé
Récapitulatif technique
1. @property --progress — propriété custom typée <number>, valeur initiale 0.
2. @starting-style { opacity: 0 } — l'élément entre invisible, transite vers son état normal.
3. animation: progress linear both; animation-timeline: view(); — le scroll drive --progress de 0 → 1.
4. opacity / transform / filter lisent --progress via var() — jamais animées directement par le scroll.

Résultat : un fade-in à l'apparition (géré par @starting-style), puis un motion de chute lié au défilement — deux mécanismes qui ne se marchent pas dessus.
