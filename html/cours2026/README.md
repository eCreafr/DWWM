# 📚 Formation HTML/CSS - Plan de cours sur 4 semaines

## 🎯 Objectif
Formation complète HTML/CSS pour adultes débutants, du niveau zéro jusqu'à la maîtrise de Flexbox, Grid et Bootstrap.

## 📋 Structure du cours

### **Semaine 1 : Fondamentaux HTML & CSS de base** ✅ COMPLET
1. [Structure d'une page HTML](semaine1-jour1-structure-page-html.html) - DOCTYPE, html, head, body, métadonnées
2. [Balises sémantiques](semaine1-jour2-balises-semantiques.html) - header, nav, main, section, article, aside, footer
3. [Texte, Listes & Liens](semaine1-jour3-texte-listes-liens.html) - Titres (h1-h6), paragraphes, listes (ul/ol), liens (a)
4. [Images, Tableaux & Formulaires](semaine1-jour4-images-tableaux-formulaires.html) - img, figure, table, form, input
5. [Introduction CSS](semaine1-jour5-introduction-css.html) - Sélecteurs, propriétés, couleurs, typographie

**📦 Projet Semaine 1 :** Page "À propos de moi" avec HTML sémantique et CSS basique

---

### **Semaine 2 : CSS approfondi & Box Model** ✅ PRINCIPAUX COURS
1. [Box Model CSS](semaine2-jour1-box-model.html) - padding, border, margin, box-sizing, display
2. [Positionnement CSS](semaine2-jour2-positionnement.html) - static, relative, absolute, fixed, sticky, z-index
3. [Responsive Design](semaine2-jour3-responsive-design.html) - Unités relatives, Media queries, Mobile First
4. Backgrounds & Pseudo-classes - background-image, gradients, :hover, :focus, :nth-child _(à venir)_
5. Pseudo-éléments & Transitions - ::before, ::after, transition, transform _(à venir)_

**📦 Projet Semaine 2 :** Page portfolio responsive avec header fixe et cartes stylisées

---

### **Semaine 3 : Flexbox & Grid** ✅ COURS ESSENTIELS
1. [Introduction Flexbox](semaine3-jour1-flexbox-introduction.html) - display: flex, flex-direction, justify-content, align-items
2. Flexbox Avancé - flex-wrap, flex-grow, flex-shrink, flex-basis, gap, order _(à venir)_
3. [Introduction CSS Grid](semaine3-jour3-grid-introduction.html) - display: grid, grid-template-columns/rows, gap, grid-template-areas
4. Grid Avancé - grid-area, auto-fill, auto-fit, responsive grids _(à venir)_
5. Flexbox + Grid combinés - Quand utiliser l'un ou l'autre, nesting _(à venir)_

**📦 Projet Semaine 3 :** Layout complexe avec Grid (header, sidebar, main, footer) + composants Flexbox

---

### **Semaine 4 : Bootstrap & Projet final** ✅ INTRODUCTION
1. [Introduction Bootstrap](semaine4-jour1-bootstrap-introduction.html) - Installation CDN, système de grille, breakpoints
2. Composants Bootstrap - navbar, cards, buttons, forms, classes utilitaires _(à venir)_
3. Bootstrap Avancé - carousel, modal, accordion, responsive utilities _(à venir)_
4. Personnalisation Bootstrap - Surcharger les styles, custom colors _(à venir)_
5. **🎓 Projet Final** - Site web complet multi-pages responsive _(à venir)_

**📦 Projet Final Semaine 4 :**
- Site web complet de 4-5 pages
- Responsive (mobile, tablette, desktop)
- Bootstrap + CSS personnalisé
- Navigation fonctionnelle
- Formulaire de contact
- Déploiement sur GitHub Pages ou Netlify

## 🚀 Comment utiliser ce cours

### 1. Commencez par l'index
Ouvrez [index.html](index.html) dans votre navigateur pour une vue d'ensemble interactive.

### 2. Suivez l'ordre recommandé
Les cours sont conçus pour être suivis dans l'ordre, chaque leçon s'appuyant sur les précédentes.

### 3. Pratiquez avec les exercices
Chaque cours se termine par un exercice pratique. **Ne passez pas à la leçon suivante sans avoir pratiqué !**

### 4. Utilisez les DevTools
Apprenez à inspecter le code avec les outils de développement de votre navigateur (F12).

### 5. Consultez la documentation
- [MDN Web Docs](https://developer.mozilla.org/fr/) - Référence complète
- [CSS-Tricks](https://css-tricks.com/) - Astuces et guides
- [Can I Use](https://caniuse.com/) - Compatibilité navigateurs
- [Bootstrap Docs](https://getbootstrap.com/docs/) - Documentation officielle

## 💡 Conseils pour réussir

### ✅ À faire
- **Pratiquer tous les jours** (même 30 minutes)
- **Coder en même temps** que vous suivez le cours
- **Expérimenter** : modifier les exemples, tester des variations
- **Lire le code** : comprendre chaque ligne
- **Poser des questions** : rechercher sur Google, Stack Overflow
- **Prendre des notes** : créer votre propre documentation

### ❌ À éviter
- Lire passivement sans coder
- Copier-coller sans comprendre
- Sauter les exercices
- Vouloir aller trop vite
- Se décourager face aux erreurs (c'est normal !)
